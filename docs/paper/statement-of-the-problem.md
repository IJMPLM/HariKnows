# Statement of the Problem

## Problem Context
The Office of the University Registrar needs a way to answer common questions, manage document requests, and keep student account information synchronized without relying only on manual office handling.

## Core Problems
1. Students often need registrar information outside office hours.
2. Document request progress is difficult to monitor when information is scattered.
3. Guest users need public guidance without access to private student records.
4. Student account credentials must be updated only for existing student numbers.
5. Bulk CSV imports need to accept real registrar formats without depending on file names.
6. Registrar staff need a clear and accurate database-backed view of student accounts.

## Research Questions
- How can a registrar helpdesk provide useful answers while protecting private student data?
- How can the system track document requests from initiation to disposal?
- How can existing student records be used to update account credentials safely?
- How can CSV uploads be parsed reliably using header structure instead of file naming conventions?
- How can the system support both public guidance and signed-in account-aware workflows?
