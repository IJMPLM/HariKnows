Registrar Helpdesk and Student Account Guide

This printable manual documents the current helpdesk, dashboard, and registrar account flows in HariKnows. It is designed to be exported to PDF from a browser print dialog or markdown-to-HTML preview.

Purpose

HariKnows provides registrar guidance, student request tracking, and account management for the Office of the University Registrar. The system separates guest guidance from account-aware student tools and registrar staff actions.

Access Model

Guests can read public guidance. Signed-in students can view their own account and request data. Registrar staff can update student credentials and manage CSV imports.

1. User Roles

Guest User

Guests can ask general registrar questions and read published FAQs and policy guidance. They do not have access to account-specific records or request history.

Signed-in Student

Signed-in students can view their profile, monitor their requests, and change their own password. Their responses are tied to the student number in the database.

Registrar Staff

Registrar staff can search student accounts, overwrite credentials for an existing student number, import ICTO CSV files, and maintain FAQ/context content for the helpdesk.

2. Student Workflow

  1. Open HariKnows in the browser and sign in with the official student email and password.
  2. Open the dashboard to view the profile card and request history.
  3. Open the account page to change the password if needed.
  4. Use the helpdesk chat to ask about documents, graduation, and office routing.

3. Registrar Workflow

  1. Open the registrar admin interface.
  2. Search the student account list by student number, name, email, or program.
  3. Select a student and overwrite only the email and password if the student number already exists.
  4. Upload an ICTO CSV file to update multiple student accounts at once.
  5. Upload FAQ/context CSV files when registrar guidance changes.
  6. Review the live account list and tables for validation.

4. Student Account Features

The current account page is intentionally focused on changing the password for the signed-in student only. Password recovery is not part of the current scope.
  • The profile page displays the student number, name, email, college, and program stored in the database.
  • The change-password action requires the current password, a new password, and confirmation.
  • The system rejects weak updates and incorrect current passwords.

5. Document Request Tracking

The request lifecycle is Pending, Prepared, Claimed, and Disposed. Students can monitor their own request status from the dashboard, but official document release still follows on-site registrar processing.

8. Troubleshooting

  • If you cannot sign in, verify the email and password on the account.
  • If a password change fails, confirm the current password and new password length.
  • If a document request is missing, confirm that the student number exists in the database.
  • If a CSV row fails, review the header names and required fields.
  • If you are a guest, account-specific information will not be available until you sign in.
